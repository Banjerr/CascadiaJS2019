# Because

* CodeMods unblock complex, application-wide refactors. For example, React used CodeMods to update their entire library to ES2015+ Class syntax. 
* CodeMods reduce tech debt (for example, no longer having the older function declaration form of creation React components, compared to the newer class syntax from the example above)
* They save tons of developer hours (for example, not having to manually update all of that code)
